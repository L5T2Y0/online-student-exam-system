# GitHub 上传步骤指南

## 📋 准备工作

### 1. 确保已安装 Git
打开命令行，输入：
```bash
git --version
```
如果没有安装，请访问：https://git-scm.com/downloads

### 2. 配置 Git（首次使用需要）
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

## 🚀 方法一：完整命令（推荐）

### 步骤1：在GitHub创建新仓库

1. 访问 https://github.com/new
2. 填写仓库信息：
   - **Repository name**: `online-exam-system`
   - **Description**: `一个功能完整、安全可靠的在线考试系统 - 支持题库管理、自动组卷、在线考试、自动评分`
   - **Public** 或 **Private**（选择公开或私有）
   - **不要**勾选 "Add a README file"
   - **不要**勾选 "Add .gitignore"
   - **不要**勾选 "Choose a license"
3. 点击 "Create repository"
4. **复制仓库地址**（例如：https://github.com/你的用户名/online-exam-system.git）

### 步骤2：在项目目录执行以下命令

打开命令行（PowerShell或CMD），进入项目目录：
```bash
cd D:\xiangmuwenjianjia\online-student-exam-system-1
```

然后依次执行以下命令：

```bash
# 1. 初始化Git仓库
git init

# 2. 添加所有文件
git add .

# 3. 提交
git commit -m "feat: 初始版本 v1.1.1 - 完整的在线考试系统

- 完整的前后端代码
- 23个测试用户（1管理员 + 2教师 + 20学生）
- 27道测试题目（7个科目）
- 5份测试试卷
- 完整的文档和测试报告
- 安全特性（JWT、CORS、频率限制、XSS防护）"

# 4. 添加远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/online-exam-system.git

# 5. 重命名分支为main
git branch -M main

# 6. 推送到GitHub
git push -u origin main
```

### 步骤3：输入GitHub凭据

如果提示输入用户名和密码：
- **用户名**: 你的GitHub用户名
- **密码**: 使用 Personal Access Token（不是GitHub密码）

#### 如何获取 Personal Access Token：
1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token" -> "Generate new token (classic)"
3. 勾选 `repo` 权限
4. 点击 "Generate token"
5. **复制生成的token**（只显示一次）
6. 在命令行密码提示时粘贴这个token

## 🚀 方法二：使用GitHub Desktop（更简单）

### 步骤1：下载并安装 GitHub Desktop
访问：https://desktop.github.com/

### 步骤2：登录GitHub账户
打开GitHub Desktop，登录你的GitHub账户

### 步骤3：添加本地仓库
1. 点击 "File" -> "Add local repository"
2. 选择项目文件夹：`D:\xiangmuwenjianjia\online-student-exam-system-1`
3. 如果提示"This directory does not appear to be a Git repository"，点击 "create a repository"

### 步骤4：发布到GitHub
1. 填写 Commit 信息：
   ```
   feat: 初始版本 v1.1.1 - 完整的在线考试系统
   ```
2. 点击 "Commit to main"
3. 点击 "Publish repository"
4. 填写仓库信息：
   - Name: `online-exam-system`
   - Description: `一个功能完整、安全可靠的在线考试系统`
   - 选择 Public 或 Private
5. 点击 "Publish Repository"

## ❓ 常见问题

### Q1: 推送时提示"Permission denied"
**解决方案**: 使用Personal Access Token代替密码

### Q2: 推送时提示"remote: Repository not found"
**解决方案**: 检查仓库地址是否正确，确保仓库已创建

### Q3: 文件太大无法上传
**解决方案**: 
- 确保 `node_modules` 被 `.gitignore` 忽略
- 检查是否有大文件（GitHub单文件限制100MB）

### Q4: 推送很慢
**解决方案**: 
- 检查网络连接
- 考虑使用国内Git镜像
- 使用GitHub Desktop可能更快
